```razor
@code {
    private string _jsonLd = string.Empty;

    protected override void OnInitialized()
    {
        var organization = new Organization
        {
            Name = "AutoDokas",
            Url = new Uri("https://autodokas.lt"),
        };

        var webSite = new WebSite
        {
            Name = "AutoDokas",
            Url = new Uri("https://autodokas.lt"),
            InLanguage = "lt",
        };

        var webPage = new WebPage
        {
            Name = "Automobilio pirkimo pardavimo sutartis internetu | AutoDokas",
            Description = "Automobilio pirkimo pardavimo sutartis internetu. Užpildykite, pasirašykite ir atsisiųskite — greitai, saugiai, be spausdintuvo.",
            Url = new Uri("https://autodokas.lt/"),
            InLanguage = "lt",
            IsPartOf = webSite,
        };

        var service = new Service
        {
            Name = "Automobilio pirkimo pardavimo sutarties sudarymas internetu",
            Description = "Internetinė paslauga automobilio pirkimo pardavimo sutarčiai sudaryti, pasirašyti ir atsisiųsti PDF formatu.",
            Url = new Uri("https://autodokas.lt/"),
            Provider = organization,
            ServiceType = "Automobilio pirkimo pardavimo sutarties sudarymas",
            AreaServed = "Lietuva",
        };

        var schemas = new ISchemaType[] { organization, webSite, webPage, service };
        var jsonParts = schemas.Select(s => s.ToHtmlEscapedString());
        _jsonLd = string.Join("\n", jsonParts.Select(j =>
            $"<script type=\"application/ld+json\">{j}</script>"));
    }
}
```

**Key implementation details:**

1. **ToHtmlEscapedString() not ToString():** Always use `ToHtmlEscapedString()` to prevent XSS when embedding in script tags. See 02-RESEARCH.md Pitfall 2.

2. **Separate script tags per schema:** Using separate `<script type="application/ld+json">` tags for each schema type. Google supports multiple JSON-LD blocks equally well as a single @graph block. Separate tags are simpler with Schema.NET since each object serializes independently with its own @context.

3. **No SearchAction on WebSite:** Google retired the sitelinks search box feature in November 2024. Do NOT add potentialAction/SearchAction. See 02-RESEARCH.md Pitfall 6.

4. **Minimal Organization schema:** Using only `Name` and `Url` since no business address, phone, or email was provided. A minimal but valid schema is better than no schema.

5. **Service schema properties:**
   - `Name`: Service name in Lithuanian
   - `Description`: What the service does in Lithuanian
   - `Provider`: Links to Organization (establishes relationship)
   - `ServiceType`: Category of service
   - `AreaServed`: "Lietuva" (Lithuania)

6. **MarkupString in page body, NOT in HeadContent:** Placing JSON-LD MarkupString in the page body avoids the MarkupString + HeadContent edge case. JSON-LD is valid anywhere in the page per Google's documentation.

**Do NOT:**
- Put `@((MarkupString)_jsonLd)` inside `<HeadContent>` (edge case with MarkupString rendering)
- Use `ToString()` instead of `ToHtmlEscapedString()` (XSS risk)
- Add SearchAction to WebSite schema (Google retired feature November 2024)
- Add `<script>` tags directly in the Razor markup (causes RZ9992 compiler error)
