**Part A: Add FAQPage JSON-LD to Home.razor @code block**

In the `OnInitializedAsync` method, after the existing schema definitions (organization, webSite, webPage, service), add the FAQPage schema:

1. Create a FAQPage object using Schema.NET:
   ```csharp
   var faqPage = new FAQPage
   {
       Name = "Dažnai užduodami klausimai apie automobilio pirkimo pardavimo sutartį",
       MainEntity = new List<IQuestion>
       {
           new Question
           {
               Name = Text.Faq1Question,
               AcceptedAnswer = new Answer { Text = StripHtml(Text.Faq1Answer) }
           },
           // ... repeat for all FAQ pairs
       }
   };
   ```

2. **IMPORTANT: MainEntity API caveat.** The exact Schema.NET 13.0.0 API for `MainEntity` on `FAQPage` may use `Values<T1, T2>` wrapper types. If `List<IQuestion>` doesn't compile for MainEntity:
   - Try: `MainEntity = new Values<ICreativeWork, IItemList>(questions.Cast<ICreativeWork>().ToList())`
   - Or try setting questions as individual property values
   - As a LAST RESORT: construct the FAQ JSON-LD as a raw JSON string and render it as a separate script tag

3. Add the faqPage to the existing schemas array:
   ```csharp
   var schemas = new Thing[] { organization, webSite, webPage, service, faqPage };
   ```

4. **HTML stripping for JSON-LD:** FAQ answers in Text.resx contain HTML (`<a href="...">` links) which should NOT appear in JSON-LD structured data. Add a simple helper method to strip HTML tags:
   ```csharp
   private static string StripHtml(string html)
   {
       return System.Text.RegularExpressions.Regex.Replace(html, "<[^>]+>", "");
   }
   ```
   Use this for JSON-LD answer text only. The Razor template still uses the HTML version via MarkupString.

5. The JSON-LD rendering pipeline is unchanged -- the existing `ToHtmlEscapedString()` + `_jsonLd` MarkupString pattern handles all schemas including the new FAQPage.
