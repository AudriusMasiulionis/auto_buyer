# Phase 2.1: Homepage Content Optimization - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Optimize the visible content on the homepage to improve search engine relevance, content depth, and quality signals. Add new content sections (FAQ, stats, service description), expand existing copy with keyword variations, fix heading hierarchy, add FAQ JSON-LD structured data, and add a site-wide footer with legal links. All changes target Home.razor, Text.resx, layout components, and existing JSON-LD code.

</domain>

<decisions>
## Implementation Decisions

### New Content Sections
- Add FAQ section with 5-8 questions — mix of contract-process questions and broader vehicle sale questions that lead to needing a contract
- FAQ displayed fully expanded (no accordion/collapsible) — all content crawlable without JS
- Add stats/counters section with real data pulled from the database (e.g., contracts created count)
- Add service description paragraph between hero and "How It Works" — 3-4 sentences explaining what AutoDokas does, who it's for, and why

### Copy & Keyword Depth
- Expand existing section body text from 1 sentence to 2-3 sentences each, naturally weaving in keyword variations ('automobilio sutartis', 'pirkimo pardavimo sutartis', 'transporto priemonės sutartis')
- Expand hero subtitle from 1 sentence to 2-3 sentences with benefit keywords ('nemokama', 'greitai', 'teisiškai galiojanti')
- Research competitor keyword usage (autosutartis.lt and top-ranking pages for 'automobilio pirkimo pardavimo sutartis') before writing copy

### Heading Structure
- Change h2 headings to include target keywords (e.g., 'Kaip sudaryti automobilio sutartį?' instead of 'Kaip tai veikia?')
- Fix heading hierarchy: change h5 step titles and feature titles to h3 (proper h1→h2→h3 flow)
- FAQ section heading should also be keyword-rich h2

### FAQ JSON-LD Schema
- Add FAQPage JSON-LD structured data alongside existing schemas (Organization, WebSite, WebPage, Service)
- Use Schema.NET FAQPage type for type-safe generation
- Enables FAQ rich snippets in Google SERP — higher click-through rate

### Internal Linking
- Add site-wide shared footer via layout component (MainLayout.razor or equivalent)
- Footer contains: links to /terms-and-conditions, /privacy-policy + copyright line ('© 2024 AutoDokas.lt')
- Include contextual links to /contract within FAQ answers where natural (micro-CTAs like 'Pradėkite čia')

### Claude's Discretion
- Exact FAQ questions and answers content (informed by competitor research)
- Exact expanded copy text for each section (informed by keyword research)
- Stats section design — which counters to show, visual presentation
- Service description paragraph wording
- Footer styling and layout
- Where to place the service description section (between hero and How It Works, or elsewhere)
- CSS adjustments needed for h5→h3 heading changes

</decisions>
