Phase 2 implemented 4 JSON-LD schemas on the homepage:
1. Organization (name: AutoDokas, url: https://autodokas.lt) -- Google rich-result eligible
2. WebSite (name: AutoDokas, url, inLanguage: lt) -- NOT rich-result eligible (valid schema.org but not in Google's gallery)
3. WebPage (name, description, url, inLanguage, isPartOf->WebSite) -- NOT rich-result eligible
4. Service (name, description, url, broker->Organization, serviceType, areaServed) -- NOT rich-result eligible

Key deviation: Service uses `Broker` instead of `Provider` because Schema.NET 13.0.0 does not expose Provider on the Service type. Broker is a valid schema.org property but less common for this use case.

**Important:** Rich Results Test only validates Google-supported rich result types. Of the 4 schemas, only Organization is eligible. If the test says "no rich results detected" for the other 3, that is expected behavior -- NOT an error. Use Schema Markup Validator (validator.schema.org) as the complementary tool for full validation of all 4 types.

import sys, re, json

print(f'Found {len(blocks)} JSON-LD blocks')
for i, block in enumerate(blocks):
    try:
        data = json.loads(block)
        schema_type = data.get('@type', 'Unknown')
        print(f'  Block {i+1}: {schema_type} - VALID JSON')
        if schema_type == 'Service':
            has_broker = 'broker' in block.lower()
            print(f'    Broker property present: {has_broker}')
    except json.JSONDecodeError as e:
        print(f'  Block {i+1}: INVALID JSON - {e}')
"
